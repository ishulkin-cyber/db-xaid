Методология оценки качества
RADPEER-адаптированная классификация · Алгоритм сравнения протоколов · Ключевые метрики
1. Как определяются протоколы
1
Первичный протокол — последняя версия отчёта, созданная врачом-описывающим (author_id = reporting_radiologist_id). Это протокол до валидации.
2
Итоговый протокол — последняя версия, созданная валидатором (author_id = validating_radiologist_id). Это эталон для сравнения.
3
В выборку попадают только задачи со статусом finalized или delivered и с заполненным validating_radiologist_id.
2. RADPEER-адаптированная классификация
GradeНазваниеОписание
1ConcordantПолное совпадение первичного и итогового протоколов.
2aMinor – StylisticДругая формулировка, без клинического значения.
2bMinor – ClinicalНаходка пропущена или недописана, не меняет тактику.
3Significant – UnderreportКлинически значимый пропуск, меняет тактику лечения.
4Significant – OverreportИзбыточное описание: находка убрана валидатором.
3. Ключевые метрики
% протоколов с пропуском
w / n × 100%
Доля задач с хотя бы одной находкой Grade 2b или 3.
Клинически значимые пропуски
count(Grade 3)
Абсолютное число находок Grade 3. Норма: ноль.
Клиническая конкордантность
(G1 + G2a) / total
Доля находок Grade 1 или 2a. Цель: > 90%.
Важно: метрики рассчитываются через Claude API (RADPEER-адаптированный грейдинг). Без ANTHROPIC_API_KEY в .env файле грейдинг недоступен — ячейки показывают приближение по числу версий протокола.
RADPEER Grade 3 = значимый клинический пропуск (меняет тактику) · Ячейка: всего / с пропуском / % DBXAID · ● Подключено
